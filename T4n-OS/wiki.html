1. Pengenalan

Tentang T4n OS

T4n OS adalah distribusi Linux berbasis Void Linux yang dioptimalkan untuk kinerja tinggi dan pengalaman pengguna yang mulus. Didesain untuk pengguna yang menginginkan sistem yang ringan, cepat, dan dapat disesuaikan.

T4n OS menggabungkan kestabilan Void Linux dengan berbagai peningkatan dan tweak khusus untuk memberikan pengalaman computing yang optimal. Sistem ini menggunakan paket manajer XBPS yang cepat dan andal, serta mendukung berbagai lingkungan desktop modern.

Catatan: T4n OS adalah proyek independen dan tidak berafiliasi dengan Void Linux secara resmi.

2. Edisi T4n OS

T4n OS tersedia dalam beberapa edisi yang dioptimalkan untuk berbagai kebutuhan dan preferensi pengguna:

XFCE Edition

Edisi XFCE menawarkan lingkungan desktop yang ringan, stabil, dan mudah digunakan. Cocok untuk pengguna yang menginginkan keseimbangan antara kinerja dan fitur.

i3 Edition

Edisi i3 menggunakan window manager tiling yang efisien untuk produktivitas maksimal. Ideal untuk pengguna yang lebih suka keyboard-driven workflow.

BSPWM Edition

Edisi BSPWM menawarkan tiling window manager yang ringan dan dapat dikonfigurasi tinggi dengan fokus pada kinerja dan minimalisme.

Hyprland Edition

Edisi Hyprland menggunakan compositor Wayland yang dinamis dengan animasi halus dan fitur modern. Cocok untuk pengguna yang menginginkan pengalaman visual yang menarik.

3. Persiapan Instalasi

Sebelum memulai instalasi T4n OS, pastikan Anda telah memenuhi persyaratan berikut:

  • Hardware: Prosesor 64-bit (x86_64), minimal 2GB RAM (4GB direkomendasikan), dan minimal 15GB ruang disk
  • Media Instalasi: USB flash drive minimal 4GB
  • Koneksi Internet: Diperlukan untuk mengunduh paket selama instalasi
  • Backup Data: Pastikan Anda telah membackup data penting sebelum instalasi

Unduh ISO T4n OS

Unduh file ISO T4n OS dari situs resmi:

https://tanos.org/download/

Verifikasi ISO

Setelah mengunduh, verifikasi integritas file ISO menggunakan checksum:

sha256sum t4n-os-*.iso

4. Pemasangan T4n OS

Proses instalasi T4n OS dirancang untuk sederhana namun fleksibel. Ikuti langkah-langkah berikut untuk menginstal sistem:

1. Boot dari Live USB

Masukkan USB flash drive yang berisi ISO T4n OS dan boot dari media tersebut. Pilih opsi "Boot T4n OS Live" dari menu boot.

2. Persiapkan Partisi

Gunakan alat partisi seperti fdisk, cfdisk, atau gparted untuk mempersiapkan partisi:

cfdisk /dev/sda

3. Mount Partisi

Mount partisi root dan home (jika ada):

mount /dev/sda2 /mnt
mkdir -p /mnt/home
mount /dev/sda3 /mnt/home

4. Instal Sistem Dasar

Jalankan instalasi dengan perintah:

t4n-installer

5. Konfigurasi Sistem

Setelah instalasi selesai, konfigurasi sistem dasar seperti pengguna, jaringan, dan driver.

6. Reboot

Keluar dari lingkungan chroot dan reboot sistem:

exit
umount -R /mnt
reboot

4.1 Live USB

Untuk membuat Live USB T4n OS, Anda dapat menggunakan salah satu metode berikut:

Menggunakan dd (Linux/macOS)

dd if=t4n-os-*.iso of=/dev/sdX bs=4M status=progress

Ganti /dev/sdX dengan device USB Anda (misal: /dev/sdb).

Menggunakan Etcher (Cross-platform)

  1. Unduh dan instal Etcher dari https://etcher.balena.io/
  2. Pilih file ISO T4n OS
  3. Pilih USB flash drive target
  4. Klik "Flash" untuk memulai proses

Menggunakan Rufus (Windows)

  1. Unduh dan instal Rufus dari https://rufus.ie/
  2. Pilih USB flash drive target
  3. Pilih opsi "DD Image" dan pilih file ISO T4n OS
  4. Klik "Start" untuk memulai proses

Peringatan: Proses ini akan menghapus semua data di USB flash drive. Pastikan Anda telah membackup data penting sebelum melanjutkan.

4.1.1 Akun Pengguna

Saat membuat Live USB, Anda perlu mengkonfigurasi akun pengguna untuk lingkungan live:

Membuat Akun Pengguna Live

# Buat user live
useradd -m -G wheel,audio,video,storage t4n
echo "t4n:password" | chpasswd

Mengatur Password Root

echo "root:password" | chpasswd

Konfigurasi Sudo

Untuk memungkinkan user t4n menggunakan sudo:

echo "%wheel ALL=(ALL:ALL) ALL" > /etc/sudoers.d/wheel

Keamanan: Password default di lingkungan live sebaiknya diganti setelah boot untuk keamanan.

4.1.2 Boot Mode

T4n OS mendukung dua mode boot: BIOS (Legacy) dan UEFI. Pastikan Anda memilih mode yang sesuai dengan sistem Anda:

Memeriksa Mode Boot

# Untuk UEFI
ls /sys/firmware/efi

# Jika direktori ada, sistem menggunakan UEFI

Boot Mode untuk Live USB

  • UEFI Mode: Pilih opsi boot dengan label "UEFI" di menu boot
  • Legacy Mode: Pilih opsi boot tanpa label "UEFI"

Implikasi Boot Mode

Mode boot yang Anda pilih akan menentukan:

  • Tabel partisi (GPT untuk UEFI, MBR untuk Legacy)
  • Lokasi bootloader (EFI partition untuk UEFI, MBR untuk Legacy)
  • Driver yang digunakan saat boot

Tip: Sebagian besar komputer modern (produksi setelah 2010) mendukung UEFI. Gunakan UEFI jika memungkinkan untuk fitur keamanan yang lebih baik.

4.2 Sistem Partisi

Partisi yang tepat sangat penting untuk kinerja dan stabilitas sistem. Berikut adalah skema partisi yang direkomendasikan untuk T4n OS:

Tipe Disk

  • Root (/): Minimal 15GB, disarankan 25-50GB untuk instalasi yang lebih lengkap
  • Home (/home): Sisa ruang disk untuk data pengguna
  • Swap: Minimal 2GB atau sama dengan ukuran RAM jika hibernasi diperlukan

Contoh Skema Partisi

/dev/sda1 - EFI System Partition (512MB, FAT32)
/dev/sda2 - Root Partition (25GB, ext4)
/dev/sda3 - Home Partition (Sisa ruang, ext4)
/dev/sda4 - Swap Partition (4GB, linux-swap)

Membuat Partisi dengan cfdisk

cfdisk /dev/sda

Dalam cfdisk:

  1. Pilih "gpt" untuk tabel partisi GPT (modern) atau "dos" untuk MBR (legacy)
  2. Buat partisi baru dengan "New"
  3. Tetapkan ukuran dan tipe partisi
  4. Tulis perubahan dengan "Write"
  5. Keluar dengan "Quit"

Format Partisi

Setelah partisi dibuat, format partisi tersebut:

mkfs.fat -F32 /dev/sda1 # EFI partition
mkfs.ext4 /dev/sda2 # Root partition
mkfs.ext4 /dev/sda3 # Home partition
mkswap /dev/sda4 # Swap partition
swapon /dev/sda4 # Activate swap

4.2.1 Tipe Disk (Root & User)

Pemilihan tipe disk yang tepat untuk partisi root dan user sangat penting untuk kinerja sistem:

Partisi Root (/)

Partisi root berisi sistem operasi dan aplikasi:

  • Ukuran minimal: 15GB
  • Ukuran disarankan: 25-50GB
  • Filesystem: ext4 (default), btrfs, atau f2fs untuk SSD
  • Mount options: defaults,noatime untuk performa lebih baik

Partisi Home (/home)

Partisi home berisi data pengguna:

  • Ukuran: Sisa ruang disk yang tersedia
  • Filesystem: ext4, btrfs (untuk snapshot), atau xfs
  • Mount options: defaults,noatime
  • Separate partition: Memudahkan reinstall tanpa kehilangan data

Format Partisi

# Format root partition
mkfs.ext4 -L ROOT /dev/sda2

# Format home partition
mkfs.ext4 -L HOME /dev/sda3

Mount Partisi

# Mount root
mount /dev/sda2 /mnt

# Buat dan mount home
mkdir /mnt/home
mount /dev/sda3 /mnt/home

Tip: Menggunakan filesystem yang sama untuk root dan home (misal ext4) menyederhanakan manajemen dan kompatibilitas.

4.2.2 Swap

Swap adalah ruang di disk yang digunakan ketika RAM penuh. Berikut adalah panduan untuk mengkonfigurasi swap:

Ukuran Swap yang Direkomendasikan

  • RAM < 2GB: Swap = 2x RAM
  • RAM 2-8GB: Swap = RAM
  • RAM > 8GB: Swap = minimal 4GB atau 8GB
  • Untuk hibernasi: Swap = RAM + sedikit buffer

Membuat Swap Partition

# Buat partition swap (misal /dev/sda4)
# Format sebagai swap
mkswap /dev/sda4

# Aktifkan swap
swapon /dev/sda4

Menggunakan Swap File

Alternatif untuk swap partition adalah swap file:

# Buat swap file 4GB
fallocate -l 4G /swapfile

# Atur permission
chmod 600 /swapfile

# Format sebagai swap
mkswap /swapfile

# Aktifkan swap
swapon /swapfile

Konfigurasi Swappiness

Swappiness mengontrol seberapa sering sistem menggunakan swap:

# Lihat nilai swappiness saat ini
cat /proc/sys/vm/swappiness

# Set swappiness ke 10 (lebih sedikit menggunakan swap)
echo "vm.swappiness=10" >> /etc/sysctl.d/99-swappiness.conf

Performa: Untuk sistem dengan RAM besar (>8GB), swap dapat dikurangi atau bahkan dihilangkan kecuali untuk hibernasi.

5. Konfigurasi Sistem

Setelah instalasi dasar selesai, Anda perlu mengkonfigurasi sistem untuk penggunaan optimal:

1. Konfigurasi Pengguna

Buat pengguna baru dan atur password:

useradd -m -G wheel,audio,video,storage username
passwd username

2. Konfigurasi Sudo

Edit file sudoers untuk memberikan akses sudo kepada grup wheel:

visudo

Hapus komentar pada baris: %wheel ALL=(ALL:ALL) ALL

3. Konfigurasi Jaringan

Untuk koneksi nirkabel, instal dan konfigurasi NetworkManager:

xbps-install -S NetworkManager
ln -s /etc/sv/NetworkManager /var/service/
sv up NetworkManager

4. Konfigurasi Locale

Atur locale untuk bahasa Indonesia:

echo "id_ID.UTF-8 UTF-8" >> /etc/locale.conf
localectl set-locale LANG=id_ID.UTF-8

5. Konfigurasi Timezone

Atur timezone ke Asia/Jakarta:

ln -sf /usr/share/zoneinfo/Asia/Jakarta /etc/localtime
hwclock --systohc

5.1 Root

Akses root di T4n OS dapat dikonfigurasi dengan beberapa cara:

Menggunakan Sudo

Cara yang direkomendasikan adalah menggunakan sudo untuk akses administratif:

sudo

Mengaktifkan Login Root Langsung

Jika Anda memerlukan login root langsung (tidak disarankan untuk keamanan):

passwd root

Mengunci Akun Root

Untuk meningkatkan keamanan, kunci akun root dan gunakan sudo saja:

passwd -l root

Keamanan: Menggunakan sudo lebih aman daripada login root langsung karena memberikan kontrol yang lebih baik atas akses administratif.

5.2 Driver

T4n OS mendukung berbagai driver hardware. Berikut adalah panduan untuk menginstal driver yang sesuai:

Driver Intel

Untuk grafis Intel:

xbps-install -S intel-media-driver intel-vaapi-driver
# Untuk Xorg
xbps-install -S xf86-video-intel

Driver AMD

Untuk grafis AMD:

xbps-install -S mesa-vulkan-radeon mesa-dri
# Untuk Xorg
xbps-install -S xf86-video-amdgpu

Driver NVIDIA

Untuk grafis NVIDIA:

# Driver open-source (nouveau)
xbps-install -S mesa-dri xf86-video-nouveau

# Driver proprietary (disarankan untuk performa maksimal)
xbps-install -S nvidia

Catatan: Setelah menginstal driver NVIDIA, Anda mungkin perlu mengkonfigurasi Xorg atau Wayland untuk menggunakan driver tersebut.

5.2.1 Driver Intel

Driver Intel untuk grafis dan media di T4n OS:

Driver Grafis Intel

# Instal driver grafis Intel
xbps-install -S mesa-dri intel-media-driver

Driver Xorg untuk Intel

# Untuk Xorg
xbps-install -S xf86-video-intel

Driver VA-API untuk Intel

# Untuk akselerasi video
xbps-install -S intel-vaapi-driver libva-intel-driver

Konfigurasi Xorg untuk Intel

Buat file konfigurasi Xorg untuk Intel:

mkdir -p /etc/X11/xorg.conf.d
echo 'Section "Device"
  Identifier "Intel Graphics"
  Driver "intel"
  Option "AccelMethod" "sna"
  Option "TearFree" "true"
EndSection' > /etc/X11/xorg.conf.d/20-intel.conf

Verifikasi Instalasi

# Periksa driver yang digunakan
glxinfo | grep "OpenGL renderer"

# Periksa akselerasi VA-API
vainfo

Tip: Untuk performa terbaik, gunakan kernel terbaru dan pastikan firmware Intel terinstal.

5.2.2 Driver AMD

Driver AMD untuk grafis di T4n OS:

Driver Grafis AMD

# Instal driver grafis AMD
xbps-install -S mesa-dri mesa-vulkan-radeon

Driver Xorg untuk AMD

# Untuk Xorg
xbps-install -S xf86-video-amdgpu

Firmware AMD

# Instal firmware AMD
xbps-install -S linux-firmware-amd

Konfigurasi Xorg untuk AMD

Buat file konfigurasi Xorg untuk AMD:

mkdir -p /etc/X11/xorg.conf.d
echo 'Section "Device"
  Identifier "AMD Graphics"
  Driver "amdgpu"
  Option "TearFree" "true"
EndSection' > /etc/X11/xorg.conf.d/20-amdgpu.conf

Verifikasi Instalasi

# Periksa driver yang digunakan
glxinfo | grep "OpenGL renderer"

# Periksa dukungan Vulkan
vulkaninfo | grep "deviceName"

Tip: Untuk kartu AMD yang lebih baru (RX series dan seterusnya), driver amdgpu sudah termasuk dalam kernel Linux terbaru.

5.2.3 Driver NVIDIA

Driver NVIDIA untuk grafis di T4n OS:

Driver Open-Source (Nouveau)

# Instal driver nouveau
xbps-install -S mesa-dri xf86-video-nouveau

Driver Proprietary (Disarankan)

# Instal driver NVIDIA proprietary
xbps-install -S nvidia

Driver NVIDIA untuk 32-bit

# Untuk aplikasi 32-bit (misal Steam)
xbps-install -S nvidia-32bit

Konfigurasi Xorg untuk NVIDIA

Buat file konfigurasi Xorg untuk NVIDIA:

mkdir -p /etc/X11/xorg.conf.d
echo 'Section "Device"
  Identifier "NVIDIA Card"
  Driver "nvidia"
  Option "NoLogo" "true"
EndSection' > /etc/X11/xorg.conf.d/20-nvidia.conf

Verifikasi Instalasi

# Periksa driver yang digunakan
glxinfo | grep "OpenGL renderer"

# Periksa dengan nvidia-settings
nvidia-settings

Mengatasi Masalah Umum

Jika mengalami masalah dengan driver NVIDIA:

# Blacklist nouveau
echo "blacklist nouveau" > /etc/modprobe.d/blacklist-nouveau.conf

# Rebuild initramfs
xbps-reconfigure -f linux

Peringatan: Driver proprietary NVIDIA mungkin tidak kompatibel dengan kernel terbaru. Gunakan kernel LTS jika mengalami masalah.

5.3 Jaringan

T4n OS menggunakan NetworkManager untuk manajemen jaringan yang mudah:

Instalasi NetworkManager

xbps-install -S NetworkManager

Aktifkan Service

ln -s /etc/sv/NetworkManager /var/service/
sv up NetworkManager

Driver Broadcom

Untuk perangkat Broadcom, instal driver dan firmware:

# Untuk chipset BCM43xx
xbps-install -S broadcom-wl-dkms

Manajemen Jaringan via CLI

Gunakan nmcli untuk manajemen jaringan via command line:

# Lihat koneksi yang tersedia
nmcli device wifi list

# Hubungkan ke jaringan WiFi
nmcli device wifi connect SSID password password

Manajemen Jaringan via GUI

Untuk lingkungan desktop, Anda dapat menggunakan applet NetworkManager:

xbps-install -S network-manager-applet

5.3.1 Driver Broadcom

Driver untuk perangkat jaringan Broadcom di T4n OS:

Identifikasi Perangkat Broadcom

# Periksa perangkat PCI
lspci | grep -i broadcom

# Periksa perangkat USB
lsusb | grep -i broadcom

Driver untuk Chipset BCM43xx

# Instal driver broadcom-wl
xbps-install -S broadcom-wl-dkms

# Muat modul kernel
modprobe wl

Driver untuk Chipset BCM4311, BCM4312, BCM4321, BCM4322

# Instal firmware
xbps-install -S linux-firmware-broadcom

# Muat modul b43
modprobe b43

Driver untuk Chipset BCM4331

# Instal driver brcmfmac
xbps-install -S linux-firmware-brcm

# Muat modul brcmfmac
modprobe brcmfmac

Mengatasi Masalah Umum

Jika driver tidak berfungsi:

# Blacklist driver yang konflik
echo "blacklist b43" >> /etc/modprobe.d/blacklist.conf
echo "blacklist ssb" >> /etc/modprobe.d/blacklist.conf

# Rebuild initramfs
xbps-reconfigure -f linux

Verifikasi Instalasi

# Periksa apakah modul dimuat
lsmod | grep -E "(wl|b43|brcmfmac)"

# Periksa interface jaringan
ip a

Tip: Beberapa laptop HP memerlukan tambahan parameter kernel. Coba tambahkan pcie_aspm=off di GRUB jika mengalami masalah.

6. Grub

GRUB (Grand Unified Bootloader) adalah bootloader default yang digunakan oleh T4n OS:

Instalasi GRUB

# Untuk sistem BIOS
grub-install --target=i386-pc /dev/sda

# Untuk sistem UEFI
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=T4n_OS

Generate Konfigurasi

grub-mkconfig -o /boot/grub/grub.cfg

Kustomisasi GRUB

Edit file /etc/default/grub untuk kustomisasi:

# Ubah timeout
GRUB_TIMEOUT=5

# Tambahkan parameter kernel
GRUB_CMDLINE_LINUX_DEFAULT="quiet loglevel=3"

Penting: Setelah mengubah /etc/default/grub, jalankan kembali grub-mkconfig untuk menerapkan perubahan.

6.1 Dual Boot

Untuk menginstal T4n OS bersama sistem operasi lain (misal Windows), ikuti langkah-langkah berikut:

Persiapan

  1. Backup data penting dari sistem yang ada
  2. Defragmentasi disk (untuk Windows)
  3. Sisakan ruang kosong untuk T4n OS (minimal 20GB)

Partisi untuk Dual Boot

# Contoh skema partisi dengan Windows
/dev/sda1 - Windows Recovery (NTFS)
/dev/sda2 - EFI System Partition (FAT32)
/dev/sda3 - Windows (NTFS)
/dev/sda4 - T4n OS Root (ext4)
/dev/sda5 - T4n OS Home (ext4)
/dev/sda6 - Swap (linux-swap)

Instalasi GRUB

Saat instalasi GRUB, pilih opsi untuk mendeteksi sistem operasi lain:

grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=T4n_OS
grub-mkconfig -o /boot/grub/grub.cfg

Memperbaiki GRUB

Jika GRUB tidak mendeteksi Windows, tambahkan manual di /etc/grub.d/40_custom:

menuentry "Windows 10" {
  insmod ntfs
  set root='(hd0,msdos3)'
  chainloader +1
}

Peringatan: Pastikan Anda memahami risiko dual boot. Kesalahan partisi dapat menyebabkan kehilangan data pada sistem yang ada.

6.2 Reinstall

Jika GRUB rusak atau tidak berfungsi, Anda dapat menginstal ulang dengan langkah-langkah berikut:

Menggunakan Live USB

  1. Boot dari Live USB T4n OS
  2. Mount partisi root sistem yang ada:
mount /dev/sda2 /mnt
  1. Mount partisi EFI (jika menggunakan UEFI):
mount /dev/sda1 /mnt/boot
  1. Masuk ke lingkungan chroot:
xbps-install -S -r /mnt grub
chroot /mnt
  1. Instal ulang GRUB:
# Untuk UEFI
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=T4n_OS
# Untuk BIOS
grub-install --target=i386-pc /dev/sda
  1. Generate konfigurasi GRUB:
grub-mkconfig -o /boot/grub/grub.cfg
  1. Keluar dari chroot dan unmount:
exit
umount -R /mnt
reboot

Memperbaiki GRUB dengan Boot-Repair

Alternatif lain adalah menggunakan Boot-Repair:

xbps-install -S boot-repair
boot-repair

7. Manajemen Paket

T4n OS menggunakan XBPS (The X Binary Package System) sebagai manajer paket utama, dengan beberapa alat tambahan untuk kemudahan penggunaan:

Fitur XBPS

  • Cepat dan efisien
  • Resolusi dependensi otomatis
  • Transaksi atomik
  • Dukungan untuk repositori pihak ketiga

Perintah Dasar XBPS

# Sinkronisasi repositori
xbps-install -S

# Instal paket
xbps-install -S nama_paket

# Hapus paket
xbps-remove -R nama_paket

# Cari paket
xbps-query -Rs nama_paket

# Perbarui sistem
xbps-install -Su

Pr0gsh

Pr0gsh adalah wrapper untuk XBPS yang menyediakan antarmuka yang lebih user-friendly:

# Instal Pr0gsh
xbps-install -S pr0gsh

# Perintah Pr0gsh
pr0gsh install nama_paket
pr0gsh remove nama_paket
pr0gsh search nama_paket
pr0gsh update

T4n-Man (t4nman)

T4n-Man adalah alat manajemen paket khusus T4n OS dengan fitur tambahan:

# Instal T4n-Man
xbps-install -S t4nman

# Perintah T4n-Man
t4nman install nama_paket
t4nman remove nama_paket
t4nman upgrade
t4nman clean

Tip: Anda dapat menggunakan salah satu alat manajemen paket ini sesuai preferensi. XBPS adalah yang paling dasar, sementara Pr0gsh dan T4n-Man menawarkan kemudahan tambahan.

7.1 XBPS

XBPS (X Binary Package System) adalah manajer paket inti dari Void Linux yang digunakan oleh T4n OS:

Perintah Dasar

# Sinkronisasi repositori
xbps-install -S

# Instal paket
xbps-install nama_paket

# Instal paket dari file
xbps-install nama_file.xbps

# Hapus paket
xbps-remove nama_paket

# Hapus paket dan dependensi yang tidak terpakai
xbps-remove -R nama_paket

# Perbarui sistem
xbps-install -Su

# Perbarui sistem dan hapus paket lama
xbps-install -Su && xbps-remove -o

Mencari Paket

# Cari paket berdasarkan nama
xbps-query -Rs nama_paket

# Cari paket berdasarkan file
xbps-query -f /path/to/file

# Lihat informasi paket
xbps-query -R nama_paket

Manajemen Repositori

# Lihat repositori yang dikonfigurasi
xbps-query -L

# Tambah repositori
xbps-install -y https://repo.example.com/current/x86_64-repodata

# Hapus repositori
xbps-remove -y repo_id

Membersihkan Cache

# Hapus cache unduhan
xbps-remove -O

# Hapus paket lama
xbps-remove -o

Menggunakan XBPS dengan Root

Sebagian besar perintah XBPS memerlukan akses root. Gunakan sudo:

sudo xbps-install -S nama_paket

7.2 Pr0gsh

Pr0gsh adalah wrapper untuk XBPS yang menyediakan antarmuka yang lebih sederhana dan user-friendly:

Instalasi

xbps-install -S pr0gsh

Perintah Dasar

# Instal paket
pr0gsh install nama_paket

# Hapus paket
pr0gsh remove nama_paket

# Perbarui sistem
pr0gsh update

# Perbarui sistem dan hapus paket lama
pr0gsh upgrade

# Cari paket
pr0gsh search nama_paket

# Lihat informasi paket
pr0gsh info nama_paket

Fitur Tambahan

  • Progress bar yang lebih informatif
  • Output yang lebih bersih dan terstruktur
  • Auto-completion untuk nama paket
  • History perintah

Konfigurasi

Pr0gsh dapat dikonfigurasi melalui file ~/.config/pr0gsh/config:

# Contoh konfigurasi
[core]
color = true
progress = true
confirm = true

Keuntungan: Pr0gsh sangat cocok untuk pengguna baru yang ingin menggunakan XBPS dengan antarmuka yang lebih sederhana.

8. Kernel

T4n OS mendukung beberapa kernel Linux yang dapat dipilih sesuai kebutuhan:

Kernel yang Tersedia

  • Linux LTS (Long Term Support) - Stabilitas maksimal
  • Linux Zen - Performa tinggi dengan optimasi desktop
  • Linux Default - Versi terbaru dengan fitur terkini

Memeriksa Kernel Saat Ini

uname -r

Menginstal Kernel Baru

# Instal kernel LTS
xbps-install -S linux-lts

# Instal kernel Zen
xbps-install -S linux-zen

# Instal kernel default
xbps-install -S linux

Menghapus Kernel

# Hapus kernel
xbps-remove -R linux-lts

Penting: Pastikan Anda memiliki setidaknya satu kernel yang berfungsi sebelum menghapus kernel yang sedang digunakan.

8.1 Kernel Linux LTS

Kernel Linux LTS (Long Term Support) adalah pilihan yang direkomendasikan untuk stabilitas dan dukungan jangka panjang:

Keuntungan Kernel LTS

  • Dukungan jangka panjang (biasanya 2-6 tahun)
  • Stabilitas tinggi dengan perbaikan bug reguler
  • Kompatibilitas hardware yang luas
  • Cocok untuk sistem produksi dan server

Instalasi

xbps-install -S linux-lts linux-lts-headers

Konfigurasi

Setelah instalasi, update GRUB untuk menambahkan kernel baru:

grub-mkconfig -o /boot/grub/grub.cfg

Reboot ke Kernel LTS

Reboot sistem dan pilih kernel LTS dari menu GRUB:

reboot

Verifikasi

uname -r

Set sebagai Default

Untuk membuat kernel LTS sebagai default, edit /etc/default/grub:

GRUB_DEFAULT=saved
GRUB_SAVEDEFAULT=true

Kemudian jalankan kembali grub-mkconfig.

Rekomendasi: Kernel LTS adalah pilihan terbaik untuk sebagian besar pengguna yang mengutamakan stabilitas daripada fitur terbaru.

8.2 Kernel Zen

Kernel Zen adalah kernel Linux yang dioptimalkan untuk performa desktop dan pengalaman pengguna yang lebih baik:

Fitur Kernel Zen

  • Optimasi untuk latensi rendah
  • Scheduling CPU yang lebih baik untuk desktop
  • Peningkatan performa I/O
  • Patch khusus untuk gaming dan multimedia

Instalasi

xbps-install -S linux-zen linux-zen-headers

Konfigurasi

Setelah instalasi, update GRUB:

grub-mkconfig -o /boot/grub/grub.cfg

Reboot ke Kernel Zen

reboot

Verifikasi

uname -r

Perbandingan dengan Kernel Lain

Fitur Kernel LTS Kernel Zen
Stabilitas Tinggi Sedang
Performa Desktop Standar Tinggi
Dukungan Hardware Luas Sedang
Dukungan Jangka Panjang Ya Tidak

Untuk Siapa: Kernel Zen cocok untuk pengguna yang menginginkan performa desktop maksimal dan tidak masalah dengan update yang lebih sering.

Berikut adalah beberapa masalah umum yang mungkin dihadapi saat menggunakan T4n OS dan solusinya:

Audio Tidak Berfungsi

Masalah XBPS

Bantuan: Jika masalah berlanjut, kunjungi forum komunitas T4n OS atau channel support untuk bantuan lebih lanjut.

9.1 Audio

Berikut adalah panduan lengkap untuk memperbaiki masalah audio di T4n OS:

Langkah 1: Periksa Hardware

# Periksa kartu audio yang terdeteksi
aplay -l

# Periksa perangkat audio
lspci -v | grep -i audio

Langkah 2: Instal Driver Audio

# Instal paket audio dasar
xbps-install -S alsa-utils pulseaudio pulseaudio-alsa

# Untuk laptop, instal tambahan
xbps-install -S sof-firmware

Langkah 3: Aktifkan Service

# Aktifkan service ALSA
ln -s /etc/sv/alsa /var/service/
sv up alsa

# Untuk PulseAudio (jika digunakan)
systemctl --user enable pulseaudio
systemctl --user start pulseaudio

Langkah 4: Periksa Volume

# Buka mixer ALSA
alsamixer

# Pastikan tidak ada yang muted (MM) dan volume cukup
# Keluar dengan ESC

Langkah 5: Test Audio

speaker-test -c 2 -l 1 -t wav

mkdir -p ~/.config/pulse

Langkah 7: Periksa Aplikasi

Beberapa aplikasi mungkin memerlukan konfigurasi tambahan:

# Untuk Firefox

Tip: Jika audio masih tidak berfungsi, coba reboot sistem setelah menginstal driver dan mengaktifkan service.

9.2 SDDM

SDDM (Simple Desktop Display Manager) adalah display manager default di T4n OS. Berikut adalah solusi untuk masalah umum:

SDDM Tidak Muncul Setelah Boot

# Periksa status service SDDM
sv status sddm

# Jika tidak aktif, aktifkan
ln -s /etc/sv/sddm /var/service/
sv up sddm

SDDM Muncul Tapi Tidak Bisa Login

# Periksa log SDDM
tail -f /var/log/sddm.log

# Periksa log Xorg
tail -f /var/log/Xorg.0.log

# Periksa permission home directory
ls -la /home/username

Error "No Session Found"

# Periksa file session
ls /usr/share/xsessions/
ls /usr/share/wayland-sessions/

# Instal desktop environment jika belum
xbps-install -S xfce4 xfce4-goodies

SDDM Crash atau Freeze

# Hapus cache SDDM
rm -rf /var/cache/sddm/*

# Restart service
sv restart sddm

# Periksa konfigurasi
cat /etc/sddm.conf

Reservasi SDDM

Jika masalah berlanjut, coba reinstall SDDM:

# Hapus SDDM
xbps-remove -R sddm

# Hapus konfigurasi residual
rm -rf /etc/sddm.conf*

# Instal ulang
xbps-install -S sddm

# Konfigurasi ulang
ln -s /etc/sv/sddm /var/service/

Menggunakan Display Manager Lain

Jika SDDM terus bermasalah, coba display manager lain:

# Instal LightDM
xbps-install -S lightdm lightdm-gtk-greeter

# Nonaktifkan SDDM
sv down sddm
rm /var/service/sddm

# Aktifkan LightDM
ln -s /etc/sv/lightdm /var/service/
sv up lightdm

Debugging: Untuk debugging lebih lanjut, jalankan SDDM secara manual dengan sddm --test-mode untuk melihat error yang terjadi.

9.3 XBPS

Berikut adalah solusi untuk masalah umum dengan XBPS (X Binary Package System):

Error "Database Locked"

# Hapus lock file
rm -f /var/db/xbps/.lock

# Coba lagi operasi XBPS
xbps-install -S

Error "Signature Verification Failed"

# Perbarui keyring
xbps-install -S void-repo-nonfree

# Atau impor manual
xbps-install -y --force https://repo.voidlinux.org/current/x86_64-repodata

Error "Package Not Found"

# Periksa ejaan nama paket
xbps-query -Rs nama_paket

# Periksa repositori yang tersedia
xbps-query -L

# Tambah repositori jika diperlukan
xbps-install -y https://repo.example.com/current/x86_64-repodata

Error "Unresolvable Dependencies"

# Perbarui sistem terlebih dahulu
xbps-install -Su

# Coba instal paket lagi
xbps-install -S nama_paket

# Jika masih gagal, coba instal dengan --force
xbps-install -S --force nama_paket

XBPS Sangat Lambat

# Ganti mirror ke yang lebih dekat
# Edit /etc/xbps.d/00-repo-main.conf
echo "repository=https://mirror.voidlinux.org/current" > /etc/xbps.d/00-repo-main.conf

# Hapus cache
xbps-remove -O

# Perbarui database
xbps-install -S

Rebuild Database XBPS

Jika database rusak parah:

# Backup database lama
cp -r /var/db/xbps /var/db/xbps.bak

# Hapus database
rm -rf /var/db/xbps/*

# Reinstall xbps
xbps-install -y --force xbps

# Sinkronisasi ulang
xbps-install -S

Tip: Sebagian besar masalah XBPS dapat diselesaikan dengan memperbarui sistem (xbps-install -Su) atau membersihkan cache (xbps-remove -O).

9.6 Partisi

Berikut adalah solusi untuk masalah umum terkait partisi di T4n OS:

Partisi Tidak Terdeteksi

# Periksa disk yang tersedia
fdisk -l

# Periksa kernel messages
dmesg | grep -i disk

# Untuk USB/NVMe, periksa modul
lsmod | grep -E '(usb|nvme)'

Filesystem Rusak

# Periksa filesystem
fsck /dev/sdaX

# Untuk perbaikan otomatis (hati-hati)
fsck -y /dev/sdaX

# Untuk filesystem ext4
e2fsck -f /dev/sdaX

Partisi Tidak Bisa Dimount

# Coba mount manual
mount /dev/sdaX /mnt

# Periksa error
dmesg | tail

# Periksa filesystem
blkid /dev/sdaX

# Jika filesystem tidak dikenali, coba format ulang
mkfs.ext4 /dev/sdaX

Ruang Disk Penuh

# Periksa penggunaan disk
df -h

# Cari file besar
find / -type f -size +100M -exec ls -lh {} \;

# Hapus cache paket
xbps-remove -O

# Hapus log lama
journalctl --vacuum-size=100M

Error "No Space Left on Device" Meskipun Ada Ruang

# Periksa inode
df -i

# Hapus file kecil banyak
find /tmp -type f -delete

# Periksa quota (jika digunakan)
quota -u username

Partisi Tidak Muncul di File Manager

# Periksa apakah sudah dimount
mount | grep /dev/sdaX

# Mount manual
sudo mount /dev/sdaX /mnt

# Untuk mount otomatis, edit /etc/fstab
echo "/dev/sdaX /mnt ext4 defaults 0 0" >> /etc/fstab

Memperbesar Partisi

Untuk memperbesar partisi (misal setelah menambah ruang disk):

# Untuk partisi yang tidak sedang digunakan
growpart /dev/sda 2

# Perbesar filesystem
resize2fs /dev/sda2

# Untuk filesystem Btrfs
btrfs filesystem resize max /mount/point

Peringatan: Operasi partisi berisiko tinggi. Selalu backup data penting sebelum melakukan operasi partisi.
